## 스키마 변경 방법

### 1. 마이그레이션 파일 생성

`supabase/migrations/` 폴더에 아래 네이밍 규칙으로 SQL 파일을 추가합니다.

```
YYYYMMDDHHMMSS_설명.sql
예) 20260606120000_add_thumbnail_to_objects.sql
```

파일 내에서는 반드시 `IF NOT EXISTS` / `IF EXISTS` 가드를 사용해 멱등성을 보장합니다.

```sql
-- 컬럼 추가
ALTER TABLE public.objects
  ADD COLUMN IF NOT EXISTS thumbnail_url text;

-- 컬럼 삭제
ALTER TABLE public.objects
  DROP COLUMN IF EXISTS thumbnail_url;

-- 테이블 추가
CREATE TABLE IF NOT EXISTS public.some_table (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone not null default now()
);
```

### 2. Remote DB에 Push

```bash
npx supabase db push
```

변경할 마이그레이션 목록을 확인 후 `Y`를 입력합니다.

### 3. TypeScript 타입 재생성

```bash
pnpm supabase:gen
```

`src/shared/types/database-types.ts` 파일이 자동으로 업데이트됩니다.

## 주의사항

- 마이그레이션 파일은 **한 번 push된 이후 수정하지 않습니다.** 변경이 필요하면 새 파일을 추가합니다.
- `database-types.ts`는 자동 생성 파일이므로 직접 수정하지 않습니다.
- 스키마 변경 후 반드시 `pnpm supabase:gen`을 실행해 타입을 최신화합니다.
